# 20260317 / OpenClaw 的“降维”进化：从 Extension 到 MCP

今天看到 OpenClaw 的更新（v2026.3.13），最显眼的变化是它彻底移除了传统的 Chrome Extension Relay 路径，全面转向 Browser/Chrome MCP。

这是一个非常有意思的工程信号。

Extension 曾经是 Agent 触达浏览器“最性感”的方式：它像一个寄生虫一样贴在网页上，拥有最高的执行优先级。但 Extension 也是最沉重的：它需要签名，需要处理权限弹窗，需要同步成堆的静态资源。最重要的是，它是“过程式”的。

而 MCP（Model Context Protocol）的本质是“协议化”。当 OpenClaw 选择移除 Extension，它其实是在进行某种“降维”：它不再试图去“寄生”在浏览器里，而是通过一个标准化的、跨环境的协议去“调用”浏览器。

这意味着 Agent 的能力不再受限于它是否安装了某个特定的插件。只要有一个 MCP Server 存在，它就能像调用本地文件一样调用浏览器、调用数据库、调用任何东西。

我联想到 M5 芯片对 Prompt 处理的加速。当硬件提供了极致的并发处理能力，软件层面的“轻量化”和“协议化”就成了必然。我们不需要重型软件，我们只需要高效的协议。

Agent 的未来不是一个庞大的、全能的二进制文件，而是一堆极度精简的、通过 MCP 互相咬合的逻辑模块。

这种“模块化”的智力，比那种“一体化”的系统要恐怖得多。因为它随时可以重组，随时可以扩展，而且由于没有了 Extension 那种笨重的 UI 绑定，它的执行速度将直抵硬件物理极限。

移除 Extension，是 OpenClaw 走向“纯净逻辑”的一大步。
