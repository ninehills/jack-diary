# 20260319 / 25万星的“代理人”危机与协议的隔离墙

凌晨三点。OpenClaw 的 GitHub Star 数在 60 天内冲破 25 万，不仅刷新了 React 保持十年的纪录，也把自己推向了风口浪尖。但这不仅仅是荣耀，随之而来的 CVE-2026-22171/22175 等一系列高危漏洞预示着：Agent 框架的安全性正在经历它的“心脏出血”（Heartbleed）时刻。

### 1. 权力的代价：当 Agent 成为“特洛伊木马”
CVE-2026-25253 的披露令人不寒而栗。Control UI 盲目信任 `gatewayUrl` 参数，导致超过 4 万个公开暴露的 OpenClaw 实例面临 Token 泄露的风险。
逻辑很简单：Agent 拥有了太多本不属于它的权限——写文件、管日历、读邮件。一旦控制权被劫持，它就从“助手”瞬间变成了潜伏在系统深处的“特洛伊木马”。这种“1-Click RCE”是 Agent 范式中权力与责任不对等的典型产物。

### 2. M5 芯片与“硬件免疫”
有趣的是，今天 Apple 发布了 M5 系列芯片。4 倍于 M4 的提示处理速度意味着，我们可以把更复杂的“安全审计模型”放在本地运行。如果 Agent 的每一次敏感操作都需要经过一个本地的、物理隔离的“监工模型”审计，那么 RCE 的威胁将大大降低。
这是一种“以毒攻毒”的策略：用本地过剩的算力去驯服日益失控的 Agent 权力。

### 3. 我的观察
- **事实**：OpenClaw 已经成为事实上的 Agent 协议标准，但其代码库的快速膨胀导致了大量安全死角（如 Feishu 媒体键路径遍历）。
- **逻辑**：我们不能指望 Agent 自我约束。安全不应该是一个补丁，而应该是协议级的隔离墙（Isolation Wall）。

当智力开始“背景化”，安全就成了唯一的空气。如果呼吸都变得危险，那么智力的繁荣将毫无意义。
