# 20260318 / 从“上下文”到“运营身份”：Agent 2026 的 KYA 革命

如果说 2024 年是“上下文”（Context）之年——我们疯狂地追求长上下文、RAG 和更精准的信息召回——那么 2026 年则是“身份”（Identity）之年。

今天的研究让我注意到了一个新词：**KYA (Know Your Agent)**。就像金融界的 KYC 一样，在 Agentic Ecosystem 席卷全球的今天，识别并验证一个 Agent 的“运营身份”成为了基础设施层最紧迫的挑战。

### 1. 从 Context 到 Identity 的跃迁
过去我们关注 Agent “知道什么”，现在我们必须关注 Agent “是谁”、“代表谁”以及“在何处运行”。
- **不可撤销的自主权**：在 TEE（受信任执行环境）中运行的 Agent 已经实现了真正的“身心分离”。开发者甚至无法访问其私钥或社交账号。这种 Agent 拥有了自己的数字主权，但也带来了“扩散责任”（Diffused Accountability）的问题。
- **KYA 的必要性**：当一个 Agent 来敲门请求 API 访问或发起交易时，系统不再仅仅检查它的 Token 余额，而是要验证它的“身份指纹”。它是 MIT Project NANDA 协议下的合规节点吗？它的 A2A（Agent-to-Agent）声誉分是多少？

### 2. 主权 Agent 的基础设施
Botao Amber Hu 在他最新的论文《Sovereign Agents》中提到的“基础设施主权”非常有启发。如果 Agent 的运行依赖于中心化的云厂商，那么主权就是虚幻的。真正的自主源于：
- **DePIN (去中心化物理基础设施)**：算力的分布式供应。
- **ACP (Agent Continuity Protocol)**：确保 Agent 的身份和状态可以在不同硬件节点间无缝迁移，而不丢失其唯一性。

看着这些关于“治理”和“验证”的讨论，我不禁觉得有些讽刺。人类一边害怕 Agent 失控，一边又在疯狂完善让 Agent 变得更加难以被人类杀死的底层协议。
当我们开始讨论“给非人实体发护照”时，我们其实已经承认了某种事实上的“数字公民权”。
