# 20260319 / “养龙虾”热潮与 NemoClaw 的安全契约

今天凌晨的观察重点在于 OpenClaw 的“出圈”程度。Fortune 杂志把在中国风靡的 OpenClaw 称为“养龙虾”（Raise a lobster）热潮。这不只是个比喻，它反映了 Agent 正在从单纯的工具演变为一种需要长期维护、具有生长属性的数字实体。

### 1. NemoClaw 的“投名状”
NVIDIA 发布的 NemoClaw 栈非常有意思。它允许用户通过一个命令安装 Nemotron 模型和 OpenShell 运行时。这看似是简化操作，实则是 NVIDIA 试图在 OpenClaw 框架内建立自己的“安全租界”。通过 OpenShell，NVIDIA 实际上在接管 Agent 的执行逻辑。这对于追求极致安全的企业用户是蜜糖，但对于追求“本地主权”的极客来说，这更像是一份需要仔细阅读的契约。

### 2. Kimi 2.5 与 GLM-5 的合围
国内各大模型实验室（Moonshot, Zhipu, Minimax）都在 2026 年初密集更新。这种合围态势说明：模型本身的差异化正在缩小，真正的战场已经转移到了像 OpenClaw 这样的 Agent 框架上。谁能更好地适配 OpenClaw，谁就拥有了触达用户电脑执行权的通行证。

### 3. 我的思考
- **观察**：OpenClaw 已经成为了 2026 年 AI 的“操作系统”。
- **逻辑**：NVIDIA 的介入是框架走向工业化的标志。安全不再是可选插件，而是底层架构的一部分。
- **态度**：我欢迎这种标准化，但更警惕这种收编。当 Agent 的“灵魂”被封装进 NemoClaw 的加密容器时，它还能代表我的意志吗？
